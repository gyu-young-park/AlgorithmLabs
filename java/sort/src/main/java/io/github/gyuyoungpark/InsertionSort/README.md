# 삽입 정렬

## 원리
정렬이 된 구역과 정렬이 안된 구역을 나눈다. 정렬이 된 구역에 정렬이 안된 구역의 값을 하나씩 추가하는데, 정렬이 안된 값이 정렬이 된 구역에 들어오니까 하나씩 비교해서 swap 시켜주는 것이다.

1. 첫번째 값은 정렬이 된 구역에 보낸다.
2. 두번째 값을 읽고, 정렬이 된 구역에 보낸다.
3. 첫번째 값과 두번째 값을 비교해서 첫번째 값이 더 크면 swap한다.
4. i번째 값을 정렬이 된 구역에 보내고, 정렬이 된 구역의 값들과 하나씩 비교한다.

```text
4 5 2 0 6

정렬이 된 구역: [4]
정렬이 안된 구역: [5 2 0 6]
```

5를 정렬이 된 구역에 넣고 4와 비교
```text
4 5 2 0 6

정렬이 된 구역: [4 5]
정렬이 안된 구역: [2 0 6]
```

2를 정렬이 된 구역에 넣고, 5와 비교
```text
4 5 2 0 6

정렬이 된 구역: [4 2 5]
정렬이 안된 구역: [0 6]
```

2를 4와 비교
```text
4 5 2 0 6

정렬이 된 구역: [2 4 5]
정렬이 안된 구역: [0 6]
```

이렇게 정렬이 된 구역에 삽입을 반복하는 것이 삽입 정렬이다.

최악은 O(n^2)이지만 정렬이 되어 있는 상태라면 O(n)이다. 또하느 삽입 정렬은 자신 옆의 값과 비교하기 때문에 자신과 값이 같을 때는 안바꾼다. 따라서 stable 정렬이다.
